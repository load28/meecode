# MeeCode

Claude Code를 더 잘 쓰기 위한 데스크탑 GUI.

---

## 왜 만들었나

Claude Code는 강력합니다. 하지만 강력함이 곧 좋은 사용자 경험을 의미하지는 않습니다.

긴 답변과 연쇄적인 도구 호출이 터미널을 빠르게 흘러가다 보면, 지금 무엇을 보고 있는지, 어디까지 진행됐는지, 방금 수정된 파일이 어디였는지 따라가기가 어렵습니다. 마크다운은 평문으로 흩어지고, 파일 경로는 클릭이 안 되고, 진행 중인 작업과 끝난 작업의 경계가 모호합니다.

CLI가 가진 단점이 아닙니다. CLI가 잘하는 일을 다르게 보여줄 화면이 필요할 뿐입니다.

**MeeCode는 그 화면입니다.** Claude Code CLI 위에 얇은 한 겹을 얹어, 같은 세션을 더 잘 읽고 더 잘 따라가게 합니다.

---

## MeeCode가 아닌 것

먼저 분명히 해 두는 편이 좋겠습니다. MeeCode는 다음 중 어느 것도 아닙니다.

- **Claude Code의 대체재가 아닙니다.** 내부에서 Claude Code CLI를 그대로 실행합니다. 모델 호출, 에이전트 루프, 세션 관리, 슬래시 커맨드, 스킬, 권한 모델 — 전부 Claude Code의 것입니다.
- **별도의 AI 에이전트가 아닙니다.** 자체 추론 로직도, 자체 도구도, 자체 메모리도 없습니다. 무언가를 더 똑똑하게 만들려 하지 않습니다.
- **새로운 워크플로우를 강제하지 않습니다.** Claude Code 사용자가 익숙한 명령과 흐름이 그대로 동작합니다.

목표는 단 하나입니다. **Claude Code를 잘 활용할 수 있는 화면을 제공하는 것.** 그 이상도 그 이하도 아닙니다.

---

## 화면이 하는 일

CLI는 한 줄씩 흘러갑니다. GUI는 공간을 씁니다. MeeCode가 공간을 쓰는 방식은 이렇습니다.

### 두 패널, 같은 흐름

왼쪽엔 압축된 질답 카드, 오른쪽엔 펼친 전체 답변. 질문 하나당 카드 하나, 클릭하면 오른쪽에서 펼쳐집니다. 스트리밍이 도중이어도 양쪽이 동시에 따라옵니다.

긴 답변을 카드 안에 다 넣으면 스크롤이 무너집니다. 짧게 자른 뒤 **더 보기**로 확장할 수 있게 했고, 깊이 읽고 싶을 땐 오른쪽 패널로 통째로 펼쳐 봅니다.

### 라이브 마크다운 스트리밍

답변이 들어오는 동안 코드블록, 표, 인용, 헤더가 **즉시** 렌더됩니다. 완료된 뒤에 평문에서 마크다운으로 점프하지 않습니다. 글자 크기가 도중에 바뀌지 않습니다.

스트리밍 토큰을 그대로 붙이면 청크 단위로 튀어 오릅니다. 그래서 글자 단위로 흘러나오는 부드러운 타자기 효과를 얹었습니다. 백로그가 쌓이면 자동으로 빨라지고, 멈추면 부드럽게 멈춥니다.

### 지금 무엇을 하고 있는가

✴ 스파클 + 동사 회전 + 점 애니메이션. VS Code Claude 확장의 진행 인디케이터와 같은 형태입니다. Cogitating, Pondering, Concocting — Claude가 무엇을 하고 있든, 한 줄로 보여줍니다.

도구를 호출 중이면 도구 이름이 그 자리에 들어옵니다. Hook이 실행 중이거나 Task subagent가 돌고 있으면 그 정보도 같은 줄에 붙습니다.

### 파일 경로는 곧 링크

`Read`, `Edit`, `Write`, `MultiEdit`, `NotebookEdit` 같은 도구 호출에서 파일 경로는 **클릭 가능한 링크**입니다. 클릭하면 별도의 코드 패널에서 해당 파일이 열리고, 코드 패널은 떼어내 별도 창으로 띄울 수도 있습니다.

읽고 있는 답변과 그 답변이 가리키는 파일이 한 화면에 같이 있어야, 답변을 검증할 수 있습니다.

### 도구 승인은 카드로

Claude가 명령을 실행하기 전, 무엇을 어떤 인자로 실행할지 카드로 먼저 보여줍니다. 허용/거부를 그 자리에서 결정하고, 인자가 마음에 들지 않으면 **그 자리에서 수정**해서 보낼 수도 있습니다.

권한 모델은 Claude Code의 것을 그대로 따릅니다. 다만 결정의 순간이 터미널 한 줄짜리 프롬프트가 아니라 카드일 뿐입니다.

### Sticky 스크롤

답변이 길어져 스크롤이 생기면 자동으로 가장 아래에 붙어 따라갑니다. 위로 올려서 이전 내용을 읽고 있을 땐 자동 스크롤이 멈춥니다. 다시 맨 아래로 가면 다시 따라갑니다.

ChatGPT, VS Code Claude 확장과 같은 패턴입니다. `ResizeObserver` 기반 이벤트 구동이라 폴링 비용도 없습니다.

## 시작하기

### 필요한 것

- Node.js 18+
- Rust toolchain (Tauri 2.x 빌드용)
- Claude Code CLI가 PATH에 설치되어 있을 것

### 개발

```bash
npm install
npm run tauri dev
```

### 빌드

```bash
npm run tauri build
```

빌드된 앱은 `src-tauri/target/release/bundle/` 아래에 있습니다.

### 테스트

```bash
npm run test
```

## 기술 스택

- **Tauri 2.x** — Rust 백엔드 + 웹뷰 프론트엔드, 시스템 통합 최소화
- **React 18 + TypeScript** — UI 레이어
- **marked + DOMPurify + Prism** — 마크다운 렌더와 코드 하이라이트
- **xterm.js** — 내장 터미널
- **react-resizable-panels** — 좌/우 패널 리사이즈

Claude Code CLI는 자식 프로세스로 실행되며, `--include-partial-messages` 모드로 SSE를 JSONL로 받아 프론트의 reducer가 세그먼트 단위로 누적합니다.
